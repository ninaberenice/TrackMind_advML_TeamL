# TrackMind AI
### AI-powered compliance intelligence for European railway certification

TrackMind reads EU TSI regulations and national rules across languages — turning 6-hour compliance searches into 20-minute cited answers with a full audit trail.

---

## What it does

Railway engineers certifying rolling stock for cross-border operation must reconcile EU Technical Specifications for Interoperability (TSIs) with national rules in languages they don't speak. A single compliance query takes 4–6 hours and costs €1,300.

TrackMind automates this:

1. **Upload** your manufacturer's technical specification (PDF) via the web interface
2. **Ask** a compliance question in any language
3. **Get** a cited verdict with confidence tier and recommended action
4. **Review** the AI draft, edit if needed, and sign off
5. **Download** a PDF audit record for the technical file

---

## Architecture

```
PDF Ingestion → Tri-Source Retrieval → LLM Reasoning → Confidence Scoring → Human Review & Audit
```

### Document sources
| Source | Language | Storage |
|--------|----------|---------|
| LOC&PAS TSI (EU 1302/2014 consolidated) | English | ChromaDB (permanent) |
| Arrêté du 19 mars 2012 (French NNTR) | French | ChromaDB (permanent) |
| Manufacturer technical specification | Any | Session memory only — never persisted |

### Key technical decisions
- **Isolated collections**: TSI, NNTR and spec are kept in separate ChromaDB collections. This forces the LLM to reason *across* source boundaries, which is what makes conflict detection possible.
- **bge-m3 embeddings**: Multilingual model from BAAI. English queries retrieve French regulatory chunks natively — no translation layer.
- **Session-based spec upload**: Manufacturer specs are never written to the vector database. They are uploaded via `/upload-spec`, chunked in memory, embedded, and stored in the server's in-memory session store (`_SESSION_SPECS`). The spec is cleared when the session ends or a new spec is uploaded.
- **Structured LLM output**: Claude Sonnet receives a system prompt encoding the EU regulatory hierarchy and returns a machine-parseable response: `VERDICT | EXPLANATION | RECOMMENDED ACTION | CONFIDENCE | CITATIONS`.
- **Human-in-the-loop**: The AI produces a draft. An engineer reviews, edits, and signs off. RED confidence tier blocks approval automatically.
- **Audit trail**: Every query, AI response and engineer decision is logged to SQLite with timestamps. Downloadable PDF audit record per decision.

## Setup

### Prerequisites
- Python 3.10+
- ~5 GB disk space (bge-m3 model + ChromaDB)
- Anthropic API key ([console.anthropic.com](https://console.anthropic.com))

### Install dependencies

```bash
python -m pip install -r requirements.txt
```

### Run the ingestion notebook (first time only)

Open `TrackMind_ingest.ipynb` and run all cells. This will:
- Auto-download the LOC&PAS TSI from EUR-Lex
- Prompt you to manually download the French NNTR from Légifrance if auto-download fails
- Chunk and embed both documents into ChromaDB (~45 minutes on CPU)
> The notebook also writes SpecDoc to ChromaDB for benchmark purposes. The live app ignores this and always uses session memory for uploaded specs.

> **Note**: The manufacturer spec (`SpecDoc.pdf`) is NOT pre-ingested. It is uploaded by the user at demo time via the `/upload-spec` endpoint. It never touches ChromaDB.


### Start the server

**Mac/Linux:**
```bash
export ANTHROPIC_API_KEY=your_key_here
uvicorn api:app --reload --port 8000
```

**Windows (PowerShell):**
```powershell
$env:ANTHROPIC_API_KEY="your_key_here"
uvicorn api:app --reload --port 8000
```

### Open the app

- **Landing page**: `http://localhost:8000/`
- **Demo interface**: `http://localhost:8000/demo.html`
- **Interactive API docs**: `http://localhost:8000/docs`

## Demo flow

1. Navigate to `http://localhost:8000/demo.html`
2. Upload `SpecDoc.pdf` (IberRail IB-EMU-450 technical specification) — the server returns a `session_id` and keeps the spec in memory only
3. Click one of the example queries or type your own
4. Wait ~1 minute for retrieval + reasoning (longer on first query while bge-m3 loads)
5. Review the verdict, explanation, citations, and retrieved chunks from all three sources
6. Edit the draft if needed, add engineer notes
7. Click **Accept Assessment** or **Reject Assessment**
8. Download the PDF audit record

## Running the benchmark

```bash
# Retrieval metrics only (no API cost)
python benchmark.py --retrieval-only

# Full benchmark (requires API key, ~8 LLM calls)
python benchmark.py

# Save results to file
python benchmark.py --save results.json

# Use mock LLM (no API cost, tests pipeline only)
python benchmark.py --mock
```

### Benchmark metrics
| Metric | Target |
|--------|--------|
| TSI retrieval precision | ≥80% |
| Cross-lingual precision (EN→FR) | ≥70% |
| Answer concordance (GREEN tier) | ≥75% |
| Confidence calibration (≥90% confidence → ≥80% correct) | ≥80% |

## API endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/health` | Server status + active session count |
| GET | `/collections` | ChromaDB collection counts (TSI + NNTR only) |
| POST | `/upload-spec` | Upload manufacturer spec PDF; returns `session_id` |
| GET | `/upload-spec/{session_id}` | Info about a loaded spec session |
| DELETE | `/upload-spec/{session_id}` | Clear a spec from session memory |
| POST | `/analyse` | Run full compliance pipeline (retrieve → reason → gate) |
| POST | `/decide` | Log engineer decision (approve / reject) |
| GET | `/audit/recent` | Recent audit entries (default n=5) |
| GET | `/audit/stats` | Audit summary statistics |

Full interactive API documentation: `http://localhost:8000/docs`

## Confidence tiers

| Tier | Confidence | Behaviour |
|------|-----------|-----------|
| GREEN | ≥90% | Draft approved for engineer review |
| AMBER | 70–89% | Draft flagged — engineer review required |
| RED | <70% | Draft blocked — escalation required, cannot be approved |

## Limitations

- Only the LOC&PAS TSI is currently pre-ingested. Other TSIs, such as INF, ENE and CCS, would require separate ingestion runs.
- Only the French NNTR is currently included. National rules from the other EU member states are not yet ingested.
- PDF parsing is optimised for structured railway technical documents. Less structured PDFs may produce noisier chunks.
- bge-m3 runs locally on CPU. The first query after server start may take around 30 seconds while the model loads.
- Uploaded manufacturer specifications are stored in memory only. Restarting the server clears all uploaded specs, so users must re-upload them.
